# C++ 기초 면접 답변

## 메모리 주소란 무엇인가요?

메모리 주소는 데이터가 저장된 메모리상의 위치를 나타내는 값입니다. 변수명은 사람이 코드를 이해하기 쉽게 붙인 이름이고, 프로그램 실행 단계에서는 컴파일러가 이를 CPU가 접근할 수 있는 형태로 변환합니다.

## 포인터를 배우기 전에 메모리 주소를 알아야 하는 이유는 무엇인가요?

포인터는 메모리 주소를 저장하는 변수이기 때문입니다. 주소의 의미를 모르면 포인터를 단순 문법으로만 외우게 되고, 동적 할당, 객체 참조, 메모리 관리 같은 개념을 이해하기 어려워집니다.

## 포인터란 무엇인가요?

포인터는 메모리 주소를 저장하는 변수입니다. C++에서는 `&` 연산자로 변수의 주소를 얻고, `*` 연산자로 포인터가 가리키는 주소의 값에 접근할 수 있습니다.

## 참조자란 무엇인가요?

참조자는 기존 변수에 붙이는 별명입니다. 포인터처럼 원본에 접근할 수 있지만, null이 될 수 없고 초기화 후 다른 대상을 참조하도록 바꿀 수 없습니다.

## 값 전달과 참조 전달의 차이는 무엇인가요?

값 전달은 인자의 값이 복사되어 함수 안으로 들어가기 때문에 함수 안에서 수정해도 원본은 바뀌지 않습니다. 참조 전달은 함수 안의 매개변수가 원본 변수의 별명이 되므로 함수 안에서 수정하면 원본도 변경됩니다.

## 스택과 힙의 차이는 무엇인가요?

스택은 함수 호출과 함께 자동으로 할당되고 함수가 끝나면 자동으로 해제되는 메모리 영역입니다. 힙은 `new` 등을 통해 동적으로 할당하는 메모리 영역이며, 개발자가 직접 해제하거나 스마트 포인터 같은 방식으로 수명을 관리해야 합니다. 스택은 빠르고 관리가 쉽지만 크기가 제한적이고, 힙은 유연하지만 메모리 누수 위험이 있습니다.

## `int* mp = new int(50);`에서 `mp`와 `new int(50)`은 각각 어디에 저장되나요?

`mp` 포인터 변수 자체는 지역 변수이므로 스택에 저장됩니다. 반면 `new int(50)`으로 생성한 실제 int 값은 힙에 저장됩니다. `delete`를 하지 않으면 힙 메모리가 해제되지 않아 메모리 누수가 발생할 수 있습니다.

## `new`와 `delete`는 무엇인가요?

`new`는 힙 영역에 메모리를 동적으로 할당하고 그 주소를 반환합니다. `delete`는 `new`로 할당한 힙 메모리를 해제합니다. 해제하지 않으면 메모리 누수가 발생할 수 있고, 해제 후 접근하면 댕글링 포인터 문제가 생길 수 있습니다.

## 댕글링 포인터란 무엇인가요?

댕글링 포인터는 이미 해제된 메모리를 여전히 가리키고 있는 포인터입니다. `delete` 이후 포인터 변수에 이전 주소값이 남아 있는 상태에서 역참조하면 위험하므로, 해제 후 `nullptr`로 초기화하는 방식으로 실수를 줄일 수 있습니다.

## 생성자와 소멸자는 무엇인가요?

생성자는 객체가 생성될 때 자동으로 호출되어 초기화 작업을 수행하는 함수입니다. 소멸자는 객체가 소멸될 때 자동으로 호출되어 동적 할당 메모리 해제나 파일 닫기 같은 자원 정리를 수행합니다. 스택 객체는 스코프를 벗어나면 소멸자가 자동 호출되고, 힙 객체는 `delete`를 호출해야 소멸자가 실행됩니다.

## RAII란 무엇인가요?

RAII는 자원의 획득과 해제를 객체의 생성자와 소멸자에 묶는 C++의 자원 관리 방식입니다. 객체가 생성될 때 자원을 획득하고, 객체가 스코프를 벗어나 소멸될 때 소멸자에서 자동으로 자원을 해제하게 만들어 메모리 누수 같은 실수를 줄일 수 있습니다. 스마트 포인터는 이 RAII 원리를 이용해 동적 메모리를 자동으로 관리해주는 도구입니다.

## 스마트 포인터란 무엇인가요?

스마트 포인터는 RAII 원리를 이용해 동적 메모리를 자동으로 관리하는 포인터입니다. `unique_ptr`은 하나의 소유자만 가지며 복사가 금지되고, 소유권 이전은 `std::move`로 가능합니다. `shared_ptr`은 참조 카운트를 통해 여러 소유자가 객체를 공유할 수 있고, `weak_ptr`은 객체를 소유하지 않고 관찰만 하며 순환 참조를 방지할 때 사용합니다.

## `unique_ptr`은 왜 복사할 수 없나요?

`unique_ptr`은 하나의 객체에 대해 유일한 소유권을 가지는 스마트 포인터입니다. 복사를 허용하면 두 `unique_ptr`이 같은 메모리를 소유하게 되어 같은 메모리를 두 번 해제할 위험이 있습니다. 그래서 복사는 금지되고, 소유권 이전은 `std::move`로만 가능합니다.

## const는 왜 사용하나요?

`const`는 값이나 객체를 수정하지 않겠다는 의도를 코드에 표현하는 키워드입니다. 변수에 사용하면 값을 변경할 수 없고, 함수 인자에서 `const T&` 형태로 사용하면 복사 비용을 줄이면서 함수 안에서 수정하지 않도록 보장할 수 있습니다. 멤버 함수 뒤에 붙는 `const`는 해당 함수가 객체의 상태를 변경하지 않는다는 의미입니다.

## `const Player& player`는 왜 사용하나요?

`const Player&`는 객체를 복사하지 않고 참조로 받아 복사 비용을 줄이면서, 함수 안에서는 해당 객체를 수정하지 않겠다는 의미입니다. 주로 큰 객체를 읽기 전용으로 전달할 때 사용합니다.

## static은 무엇인가요?

`static`은 위치에 따라 의미가 조금 다르지만, 공통적으로 객체 수명이나 객체별 소유와 다른 방식으로 값을 유지하거나 공유할 때 사용합니다. 지역 변수에 붙이면 함수가 끝나도 값이 유지되고, 클래스 멤버 변수에 붙이면 객체마다 따로 생성되지 않고 클래스 단위로 하나만 존재합니다. `static` 멤버 함수는 객체 없이 호출할 수 있지만, 특정 객체의 일반 멤버 변수에는 직접 접근할 수 없습니다.

## static 지역 변수는 일반 지역 변수와 무엇이 다른가요?

일반 지역 변수는 함수가 호출될 때마다 새로 생성되고 함수가 끝나면 사라집니다. 반면 `static` 지역 변수는 처음 한 번만 초기화되고 함수가 끝난 뒤에도 값이 유지되어, 다음 호출 때 이전 값을 이어서 사용할 수 있습니다.

## virtual 함수란 무엇인가요?

`virtual` 함수는 부모 클래스 포인터나 참조로 자식 객체를 다룰 때, 실제 객체 타입에 맞는 재정의 함수를 호출하게 해주는 기능입니다. 이를 통해 런타임 다형성을 구현할 수 있습니다.

## virtual과 override의 차이는 무엇인가요?

`virtual`은 부모 클래스에서 이 함수가 자식에서 재정의될 수 있고, 부모 포인터나 참조로 호출해도 실제 객체 타입의 함수를 실행하라는 의미입니다. `override`는 자식 클래스에서 부모의 `virtual` 함수를 올바르게 재정의했는지 컴파일러가 확인하게 해주는 키워드입니다.

## 일반 virtual 함수와 순수가상함수는 무엇이 다른가요?

일반 `virtual` 함수는 부모가 기본 구현을 가질 수 있고, 자식이 재정의하지 않으면 부모 구현이 호출됩니다. 순수가상함수는 `virtual void Attack() = 0;`처럼 선언하며, 자식 클래스가 반드시 구현해야 합니다. 순수가상함수를 가진 클래스는 추상 클래스가 되어 직접 객체를 생성할 수 없습니다.

## 부모 소멸자는 왜 virtual로 만들어야 하나요?

부모 포인터로 자식 객체를 삭제할 수 있는 구조라면 부모 소멸자를 `virtual`로 만들어야 합니다. 그렇지 않으면 `delete` 시 자식 소멸자가 제대로 호출되지 않아 자원 정리가 누락될 수 있습니다.
