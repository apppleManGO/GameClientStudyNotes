# 참조자

## 한 줄 정의

참조자는 기존 변수에 붙이는 또 다른 이름입니다.

## 왜 필요한가

원본 값을 복사하지 않고 직접 다루고 싶을 때 사용합니다.

포인터처럼 원본에 접근할 수 있지만, 문법적으로는 일반 변수처럼 사용할 수 있어서 더 안전하고 읽기 쉽습니다.

## 핵심 설명

```cpp
int hp = 100;
int& ref = hp;
```

`ref`는 새로운 값을 따로 가진 변수가 아니라 `hp`의 별명입니다.

```cpp
ref = 40;
```

위 코드를 실행하면 `hp`도 40이 됩니다.

```text
hp와 ref는 같은 메모리 공간을 가리키는 두 이름
```

## 포인터와 차이

```text
포인터
- 주소를 저장하는 변수
- nullptr 가능
- 다른 대상을 다시 가리킬 수 있음
- 사용할 때 *ptr처럼 역참조가 필요함

참조자
- 기존 변수의 별명
- null 불가능
- 처음 연결한 대상을 바꿀 수 없음
- 일반 변수처럼 사용함
```

## 함수와 참조자

일반 인자로 전달하면 값이 복사됩니다.

```cpp
void Damage(int hp)
{
    hp -= 10;
}

int playerHp = 100;
Damage(playerHp);
```

이 경우 함수 안의 `hp`는 `playerHp`의 복사본입니다.

따라서 함수 안에서 `hp`를 바꿔도 원본 `playerHp`는 바뀌지 않습니다.

참조자로 받으면 함수 안의 `hp`가 바깥의 `playerHp`의 별명이 됩니다.

```cpp
void Damage(int& hp)
{
    hp -= 10;
}

int playerHp = 100;
Damage(playerHp);
```

이 경우 `playerHp`는 90으로 변경됩니다.

## 값 전달, 포인터 전달, 참조 전달

```cpp
void DamageByValue(int hp)
{
    hp -= 10;
}
```

값이 복사됩니다. 원본은 바뀌지 않습니다.

```cpp
void DamageByPointer(int* hp)
{
    *hp -= 10;
}
```

주소를 받아서 원본 값을 수정합니다.

```cpp
void DamageByReference(int& hp)
{
    hp -= 10;
}
```

원본의 별명으로 받아서 원본 값을 수정합니다.

## const 참조자

객체가 클 때는 복사 비용을 줄이기 위해 참조자로 받는 경우가 많습니다.

하지만 함수 안에서 수정하면 안 되는 경우도 있습니다.

```cpp
void PrintPlayerInfo(const Player& player)
{
    // player를 복사하지 않고 읽기만 함
}
```

`const Player&`는 복사하지 않고 원본을 참조하되, 함수 안에서 수정하지 않겠다는 뜻입니다.

## 면접 답변

참조자는 기존 변수에 붙이는 별명입니다. 포인터처럼 원본에 접근할 수 있지만, null이 될 수 없고 초기화 후 다른 대상을 참조하도록 바꿀 수 없습니다. 함수 인자로 사용할 때 원본을 수정하거나, 큰 객체를 복사하지 않고 전달할 때 자주 사용합니다.
