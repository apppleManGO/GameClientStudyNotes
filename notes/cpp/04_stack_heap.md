# 스택과 힙

## 한 줄 정의

스택은 자동으로 관리되는 메모리이고, 힙은 개발자가 직접 수명을 관리해야 하는 메모리입니다.

## 왜 필요한가

C++에서는 변수를 만들 때 그 값이 어디에 만들어지고, 언제 사라지는지를 이해해야 합니다.

특히 포인터, 동적 할당, 메모리 누수, 스마트 포인터, Unreal의 객체 관리까지 모두 스택과 힙 개념에서 이어집니다.

## 스택

스택은 함수가 실행될 때 자동으로 사용되는 메모리입니다.

```cpp
void Test()
{
    int hp = 100;
}
```

`hp`는 지역 변수이므로 보통 스택에 생성됩니다.

함수가 끝나면 자동으로 제거됩니다.

```text
함수 시작
-> 지역 변수 생성
-> 함수 종료
-> 지역 변수 자동 제거
```

스택은 빠르고 자동으로 관리되지만, 크기가 제한적입니다.

재귀 함수가 끝없이 호출되거나 너무 큰 데이터를 스택에 만들면 스택 오버플로우가 발생할 수 있습니다.

## 힙

힙은 개발자가 직접 요청해서 사용하는 메모리입니다.

```cpp
int* mp = new int(50);
```

이 코드는 힙에 `int` 값을 하나 만들고, 그 주소를 `mp` 포인터에 저장합니다.

힙에 만든 데이터는 함수가 끝나도 자동으로 사라지지 않습니다.

따라서 직접 해제해야 합니다.

```cpp
delete mp;
```

## 중요한 예시

```cpp
void Test()
{
    int hp = 100;
    int* mp = new int(50);
}
```

이 코드에서 메모리 위치는 이렇게 나뉩니다.

```text
스택
- hp = 100
- mp = 힙에 있는 int의 주소

힙
- int 값 50
```

`hp`는 스택에 만들어지고 함수가 끝나면 자동으로 제거됩니다.

`mp` 포인터 변수 자체도 지역 변수이므로 스택에 만들어집니다.

하지만 `new int(50)`으로 만든 실제 `int` 값은 힙에 만들어집니다.

함수가 끝나면 스택에 있던 `mp`도 사라지기 때문에, 힙에 있는 `int(50)`의 주소를 잃어버립니다.

그런데 `delete`를 하지 않았으므로 힙 메모리는 계속 남습니다. 이것이 메모리 누수입니다.

## 해결 방법

직접 해제하려면 `delete`를 사용합니다.

```cpp
void Test()
{
    int* mp = new int(50);

    delete mp;
}
```

다만 현대 C++에서는 직접 `new/delete`를 많이 쓰기보다 RAII와 스마트 포인터를 선호합니다.

```cpp
auto mp = std::make_unique<int>(50);
```

스마트 포인터는 뒤에서 별도로 정리합니다.

## 스택과 힙 비교

```text
스택
- 지역 변수 저장
- 함수가 끝나면 자동 해제
- 빠름
- 크기가 제한적
- 직접 delete 필요 없음

힙
- new로 동적 할당
- 직접 delete 또는 스마트 포인터로 관리
- 스택보다 상대적으로 느림
- 큰 데이터나 오래 살아야 하는 객체에 사용
- 메모리 누수 위험 있음
```

## 게임 개발 연결

게임에서는 객체가 자주 생성되고 사라집니다.

```text
몬스터 생성
총알 생성
이펙트 생성
아이템 드롭
UI 팝업 생성
```

이런 객체를 무분별하게 힙에 만들고 지우면 성능 비용과 메모리 관리 문제가 생길 수 있습니다.

그래서 다음 개념들과 연결됩니다.

```text
Object Pooling
Garbage Collection
스마트 포인터
RAII
Unreal GC
Unity GC
```

## Unity 연결

Unity C#에서는 객체가 주로 힙에 만들어지고, 사용하지 않는 객체는 GC가 정리합니다.

하지만 객체를 계속 생성하고 버리면 GC 부담으로 프레임 드랍이 생길 수 있습니다.

그래서 총알, 이펙트 같은 반복 생성 객체에는 오브젝트 풀링을 자주 사용합니다.

## Unreal 연결

Unreal C++에서는 `UObject` 계열은 Unreal의 GC 시스템이 관리하는 경우가 많습니다.

하지만 일반 C++ 객체는 직접 수명 관리, RAII, 스마트 포인터 같은 개념을 이해해야 합니다.

## 면접 답변

스택은 함수 호출과 함께 자동으로 할당되고 함수가 끝나면 자동으로 해제되는 메모리 영역입니다. 힙은 `new` 등을 통해 동적으로 할당하는 메모리 영역이며, 개발자가 직접 해제하거나 스마트 포인터 같은 방식으로 수명을 관리해야 합니다. 스택은 빠르고 관리가 쉽지만 크기가 제한적이고, 힙은 유연하지만 메모리 누수 위험이 있습니다.
