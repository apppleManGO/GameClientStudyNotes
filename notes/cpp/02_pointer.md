# 포인터

## 한 줄 정의

포인터는 메모리 주소를 저장하는 변수입니다.

## 왜 필요한가

변수의 값을 직접 복사하지 않고, 그 값이 저장된 위치를 통해 원본에 접근하기 위해 사용합니다.

게임 개발에서는 객체가 크고 복잡한 경우가 많습니다. 플레이어, 몬스터, 컴포넌트 같은 객체를 매번 통째로 복사하기보다 주소를 통해 접근하는 방식이 더 자연스럽고 효율적입니다.

## 핵심 설명

```cpp
int hp = 100;
int* ptr = &hp;
```

위 코드는 이렇게 이해하면 됩니다.

```text
hp      -> 값 100을 가진 변수
&hp     -> hp가 저장된 메모리 주소
ptr     -> hp의 주소를 저장한 포인터 변수
*ptr    -> ptr이 가리키는 주소에 있는 값
```

즉 `ptr`은 값 100 자체를 저장하는 것이 아니라, `hp`가 저장된 위치를 저장합니다.

## `&`와 `*`

```cpp
&hp
```

`hp`의 주소를 가져옵니다.

```cpp
int* ptr
```

`ptr`이 `int` 타입 변수의 주소를 저장하는 포인터라는 뜻입니다.

```cpp
*ptr
```

`ptr`이 들고 있는 주소를 따라가서 실제 값에 접근합니다. 이것을 역참조라고 합니다.

## 값 변경 예시

```cpp
int hp = 100;
int* ptr = &hp;

*ptr = 30;
```

`ptr`은 `hp`의 주소를 저장하고 있고, `*ptr`은 그 주소에 있는 실제 값에 접근합니다.

따라서 `*ptr = 30;`을 실행하면 `hp`의 값도 30으로 변경됩니다.

## 함수와 포인터

일반 변수로 함수에 값을 넘기면 값이 복사됩니다.

```cpp
void Damage(int hp)
{
    hp -= 10;
}

int playerHp = 100;
Damage(playerHp);
```

이 경우 함수 안의 `hp`는 `playerHp`의 복사본이기 때문에 `playerHp`는 그대로 100입니다.

원본을 바꾸려면 주소를 넘길 수 있습니다.

```cpp
void Damage(int* hp)
{
    *hp -= 10;
}

int playerHp = 100;
Damage(&playerHp);
```

`&playerHp`로 주소를 넘기고, 함수 안에서 `*hp`로 그 주소에 있는 원본 값을 수정합니다.

## Unreal 연결

Unreal C++에서는 포인터가 매우 자주 등장합니다.

```cpp
AActor* Target;
UCameraComponent* Camera;
UStaticMeshComponent* Mesh;
```

이는 객체 자체를 복사해서 들고 있는 것이 아니라, 객체가 있는 위치를 가리킨다고 이해하면 좋습니다.

## 주의할 점

포인터는 아무것도 가리키지 않을 수 있습니다.

```cpp
int* ptr = nullptr;
```

`nullptr` 상태의 포인터를 역참조하면 문제가 생길 수 있습니다.

```cpp
*ptr = 10; // 위험
```

포인터를 사용할 때는 항상 "이 포인터가 지금 유효한 대상을 가리키고 있는가?"를 생각해야 합니다.

## 면접 답변

포인터는 메모리 주소를 저장하는 변수입니다. C++에서는 `&` 연산자로 변수의 주소를 얻고, `*` 연산자로 포인터가 가리키는 주소의 값에 접근할 수 있습니다. 원본 값을 수정하거나 객체를 복사하지 않고 다룰 때 사용합니다.
