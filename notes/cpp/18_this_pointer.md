# this 포인터

## 한 줄 정의

`this`는 현재 멤버 함수를 호출한 객체 자기 자신을 가리키는 포인터입니다.

## 왜 필요한가

같은 클래스의 멤버 함수 코드는 여러 객체가 함께 사용합니다.

그런데 같은 함수 코드가 실행되더라도 어떤 객체가 호출했는지에 따라 다른 멤버 변수를 사용해야 합니다.

이때 현재 함수를 호출한 객체를 가리키는 포인터가 `this`입니다.

## 기본 예시

```cpp
class Player
{
public:
    int hp;

    void SetHp(int hp)
    {
        this->hp = hp;
    }
};
```

여기서 이름이 둘 다 `hp`입니다.

```cpp
this->hp
```

는 객체의 멤버 변수 `hp`입니다.

```cpp
hp
```

는 함수 매개변수 `hp`입니다.

따라서 다음 코드는 이렇게 읽을 수 있습니다.

```cpp
this->hp = hp;
```

```text
이 객체의 hp에
매개변수 hp 값을 넣어라.
```

## 객체마다 this가 가리키는 대상이 다르다

```cpp
Player player1;
Player player2;

player1.SetHp(100);
player2.SetHp(50);
```

`player1.SetHp(100)`이 호출될 때 `this`는 `player1`을 가리킵니다.

`player2.SetHp(50)`이 호출될 때 `this`는 `player2`를 가리킵니다.

그래서 같은 `SetHp()` 함수 코드라도 어느 객체가 호출했는지에 따라 다른 객체의 `hp`를 수정합니다.

```text
this = 현재 객체의 주소
this->hp = 현재 객체의 hp
```

## 언제 자주 쓰나

`this`는 다음 상황에서 자주 볼 수 있습니다.

```text
멤버 변수와 매개변수 이름이 같을 때 구분
현재 객체의 주소를 넘길 때
현재 객체 자신을 반환할 때
```

예를 들어 멤버 변수와 매개변수 이름이 다르면 `this`를 생략해도 됩니다.

```cpp
void SetHp(int value)
{
    hp = value;
}
```

하지만 이름이 같으면 `this`를 사용해 구분하는 것이 명확합니다.

```cpp
void SetHp(int hp)
{
    this->hp = hp;
}
```

## 면접 답변

`this`는 현재 멤버 함수를 호출한 객체 자기 자신을 가리키는 포인터입니다. 멤버 변수와 매개변수 이름이 같을 때 구분하거나, 현재 객체의 주소를 반환하거나 전달할 때 사용할 수 있습니다. 같은 멤버 함수 코드라도 어떤 객체가 호출했는지에 따라 `this`가 가리키는 대상이 달라집니다.
