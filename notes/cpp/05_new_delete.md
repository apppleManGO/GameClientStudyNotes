# 동적 할당 new/delete

## 한 줄 정의

`new`는 힙에 메모리를 동적으로 할당하고, `delete`는 `new`로 할당한 힙 메모리를 해제합니다.

## 왜 필요한가

프로그램 실행 중에 필요한 만큼 메모리를 직접 만들고 싶을 때 동적 할당을 사용합니다.

예를 들어 실행 중에 몬스터 수, 아이템 수, 맵 데이터 크기 등이 결정될 수 있습니다.

이런 경우 스택에 고정된 크기로 만들기보다 힙에 필요한 만큼 만들 수 있습니다.

## 기본 사용법

```cpp
int* hp = new int(100);
```

이 코드는 이렇게 이해하면 됩니다.

```text
힙에 int 하나를 만들고 100으로 초기화한다.
그 힙 메모리의 주소를 hp 포인터에 저장한다.
```

`new`로 만든 메모리는 함수가 끝나도 자동으로 사라지지 않습니다.

따라서 직접 해제해야 합니다.

```cpp
delete hp;
```

## 메모리 누수

```cpp
void Test()
{
    int* hp = new int(100);
}
```

위 코드에서 함수가 끝나면 `hp` 포인터 변수는 사라집니다.

하지만 `new int(100)`으로 만든 힙 메모리는 자동으로 사라지지 않습니다.

게다가 주소를 저장하던 `hp`도 사라졌기 때문에 나중에 해제할 방법도 잃게 됩니다.

이런 상태를 메모리 누수라고 합니다.

## 올바른 해제

```cpp
void Test()
{
    int* hp = new int(100);

    delete hp;
}
```

`delete hp;`는 `hp` 포인터 변수를 삭제하는 것이 아니라, `hp`가 가리키고 있는 힙 메모리를 해제하는 것입니다.

## 댕글링 포인터

```cpp
int* value = new int(10);
delete value;
*value = 20;
```

`delete value;` 이후 `value`가 가리키던 힙 메모리는 해제되었습니다.

하지만 포인터 변수 `value` 안에는 이전 주소값이 남아 있을 수 있습니다.

이 상태에서 `*value = 20;`처럼 접근하면 이미 해제된 메모리에 접근하게 됩니다.

이런 포인터를 댕글링 포인터라고 합니다.

```text
댕글링 포인터 = 이미 해제된 메모리를 가리키는 포인터
```

보통 해제 후에는 `nullptr`로 바꿔둡니다.

```cpp
delete value;
value = nullptr;
```

## 배열 동적 할당

하나짜리 객체는 `new`와 `delete`를 사용합니다.

```cpp
int* hp = new int(100);
delete hp;
```

배열은 `new[]`와 `delete[]`를 사용합니다.

```cpp
int* arr = new int[5];
delete[] arr;
```

짝을 맞춰야 합니다.

```text
new   -> delete
new[] -> delete[]
```

## 현대 C++에서의 방향

`new/delete`는 C++ 메모리 관리를 이해하기 위해 꼭 알아야 합니다.

하지만 실무에서는 직접 `new/delete`를 많이 쓰기보다 RAII와 스마트 포인터를 선호합니다.

직접 관리하면 다음 문제가 생기기 쉽습니다.

```text
delete를 까먹음 -> 메모리 누수
delete 후 접근 -> 댕글링 포인터
두 번 delete -> 위험
```

그래서 뒤에서 다음 개념으로 이어집니다.

```text
RAII
std::unique_ptr
std::shared_ptr
std::weak_ptr
```

## 게임 개발 연결

게임에서는 객체가 자주 생성되고 사라집니다.

```text
총알
몬스터
이펙트
드롭 아이템
UI
```

이를 매번 `new/delete`로 만들고 지우면 성능 부담과 메모리 관리 문제가 생길 수 있습니다.

그래서 오브젝트 풀링, 스마트 포인터, Unreal GC, Unity GC 같은 개념이 중요해집니다.

## 면접 답변

`new`는 힙 영역에 메모리를 동적으로 할당하고 그 주소를 반환합니다. `delete`는 `new`로 할당한 힙 메모리를 해제합니다. `new`로 할당한 메모리를 해제하지 않으면 메모리 누수가 발생할 수 있고, 해제 후 접근하면 댕글링 포인터 문제가 생길 수 있습니다.

- `new[]`로 만든 배열은 무엇으로 해제해야 할까?
- 댕글링 포인터는 어떤 상태를 말할까?
