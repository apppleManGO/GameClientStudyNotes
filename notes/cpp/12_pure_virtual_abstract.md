# 순수가상함수와 추상 클래스

## 한 줄 정의

순수가상함수는 부모 클래스에서 구현을 제공하지 않고, 자식 클래스가 반드시 구현하도록 강제하는 함수입니다.

## 왜 필요한가

부모 클래스가 공통 규칙만 정하고, 실제 동작은 자식 클래스가 반드시 구현하게 만들고 싶을 때 사용합니다.

예를 들어 모든 몬스터는 공격할 수 있어야 하지만, 공격 방식은 몬스터마다 다를 수 있습니다.

```text
Goblin -> 칼 공격
Boss   -> 광역 공격
Slime  -> 몸통 박치기
```

이럴 때 부모 클래스는 이렇게 말할 수 있습니다.

```text
몬스터라면 Attack 함수는 반드시 있어야 한다.
하지만 구체적인 공격 방식은 자식 클래스가 정해라.
```

## 순수가상함수

순수가상함수는 함수 선언 뒤에 `= 0`을 붙입니다.

```cpp
class Monster
{
public:
    virtual void Attack() = 0;
};
```

이 뜻은 다음과 같습니다.

```text
이 함수는 부모 클래스에서 구현하지 않겠다.
자식 클래스가 반드시 구현해야 한다.
```

## 추상 클래스

순수가상함수를 하나라도 가진 클래스는 추상 클래스가 됩니다.

```cpp
class Monster
{
public:
    virtual void Attack() = 0;
};
```

추상 클래스는 직접 객체를 만들 수 없습니다.

```cpp
Monster monster; // 오류
```

이유는 `Attack()`이 아직 구현되지 않은 미완성 클래스이기 때문입니다.

```text
추상 클래스 = 직접 객체를 만들 수 없는 미완성 설계도
```

## 자식 클래스도 반드시 구현해야 한다

순수가상함수를 가진 클래스를 상속받은 자식 클래스는 해당 함수를 반드시 구현해야 합니다.

```cpp
class Slime : public Monster
{
};
```

위 코드에서 `Slime`은 `Monster`를 상속받았지만 `Attack()`을 구현하지 않았습니다.

따라서 `Slime`도 여전히 추상 클래스입니다.

```cpp
Slime slime; // 오류
```

가능하게 하려면 `Attack()`을 구현해야 합니다.

```cpp
class Slime : public Monster
{
public:
    void Attack() override
    {
        cout << "Slime Attack" << endl;
    }
};
```

이제는 객체를 만들 수 있습니다.

```cpp
Slime slime; // 가능
```

정리하면 다음과 같습니다.

```text
부모에 순수가상함수가 있음
-> 자식은 그 함수를 반드시 구현해야 함
-> 구현 안 하면 자식도 추상 클래스
-> 구현하면 객체 생성 가능
```

## 부모 포인터로 다룰 수 있다

추상 클래스는 직접 객체를 만들 수는 없지만, 포인터나 참조 타입으로는 사용할 수 있습니다.

```cpp
Monster* enemy = new Slime();
enemy->Attack();
```

이때 실제 객체는 `Slime`이고, `Attack()`은 `Slime`에서 구현한 함수가 호출됩니다.

## 일반 virtual 함수와 차이

일반 `virtual` 함수는 부모가 기본 구현을 가질 수 있습니다.

```cpp
virtual void Attack()
{
    cout << "Monster Attack" << endl;
}
```

```text
부모가 기본 구현을 제공함
자식이 재정의해도 됨
자식이 재정의하지 않으면 부모 함수 실행
```

순수가상함수는 부모가 기본 구현을 제공하지 않습니다.

```cpp
virtual void Attack() = 0;
```

```text
부모가 기본 구현을 제공하지 않음
자식이 반드시 구현해야 함
클래스를 직접 객체로 만들 수 없음
```

## 인터페이스처럼 사용

C++에는 C#처럼 `interface` 키워드가 따로 없지만, 순수가상함수만 가진 클래스를 인터페이스처럼 사용할 수 있습니다.

```cpp
class IDamageable
{
public:
    virtual ~IDamageable() = default;
    virtual void TakeDamage(int damage) = 0;
};
```

게임에서는 이런 식으로 생각할 수 있습니다.

```text
IDamageable  -> 데미지를 받을 수 있는 대상
IInteractable -> 상호작용 가능한 대상
IMovable     -> 움직일 수 있는 대상
```

## 소멸자는 virtual로

추상 클래스를 부모 타입으로 사용할 거라면 소멸자는 보통 `virtual`로 둡니다.

```cpp
class Monster
{
public:
    virtual ~Monster() = default;
    virtual void Attack() = 0;
};
```

부모 포인터로 자식 객체를 삭제할 때 자식 소멸자가 제대로 호출되게 하기 위해서입니다.

## 면접 답변

순수가상함수는 `virtual void Func() = 0;`처럼 선언하며, 부모 클래스에서 구현을 제공하지 않고 자식 클래스가 반드시 구현하도록 강제하는 함수입니다. 순수가상함수를 하나라도 가진 클래스는 추상 클래스가 되어 직접 객체를 생성할 수 없습니다. 이를 상속받은 자식 클래스도 해당 함수를 구현하지 않으면 여전히 추상 클래스입니다. 주로 공통 규칙이나 인터페이스를 정의하고, 실제 동작은 자식 클래스에서 구현하게 할 때 사용합니다.
