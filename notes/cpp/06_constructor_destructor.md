# 생성자와 소멸자

## 한 줄 정의

생성자는 객체가 만들어질 때 자동으로 호출되고, 소멸자는 객체가 사라질 때 자동으로 호출됩니다.

## 왜 필요한가

객체가 시작할 때 필요한 초기화 작업을 하고, 객체가 끝날 때 필요한 정리 작업을 하기 위해 사용합니다.

생성자와 소멸자는 뒤에서 배우는 RAII의 기반이 됩니다.

## 생성자

생성자는 객체가 만들어질 때 자동으로 호출되는 함수입니다.

```cpp
class Player
{
public:
    Player()
    {
        hp = 100;
    }

    int hp;
};
```

객체를 만들면 생성자가 자동으로 호출됩니다.

```cpp
Player player;
```

생성자는 보통 이런 일을 합니다.

```text
객체의 초기값 설정
필요한 자원 준비
객체가 정상 상태로 시작하게 만들기
```

## 생성자의 특징

생성자는 클래스 이름과 같은 이름을 가집니다.

```cpp
class Player
{
public:
    Player()
    {
    }
};
```

반환 타입이 없습니다. `void`도 쓰지 않습니다.

```cpp
Player();      // 맞음
void Player(); // 틀림
```

## 매개변수가 있는 생성자

생성자는 매개변수를 받을 수 있습니다.

```cpp
class Player
{
public:
    Player(int startHp)
    {
        hp = startHp;
    }

    int hp;
};
```

```cpp
Player player(150);
```

이 경우 `hp`는 150으로 초기화됩니다.

## 소멸자

소멸자는 객체가 사라질 때 자동으로 호출되는 함수입니다.

```cpp
class Player
{
public:
    ~Player()
    {
    }
};
```

소멸자는 클래스 이름 앞에 `~`가 붙습니다.

```text
Player()  -> 생성자
~Player() -> 소멸자
```

소멸자는 보통 이런 일을 합니다.

```text
동적 할당한 메모리 해제
파일 닫기
네트워크 연결 종료
자원 정리
```

## 호출 흐름

```cpp
#include <iostream>
using namespace std;

class Player
{
public:
    Player()
    {
        cout << "생성" << endl;
    }

    ~Player()
    {
        cout << "소멸" << endl;
    }
};

void Test()
{
    Player player;
}
```

`Test()`가 호출되면 다음 순서로 실행됩니다.

```text
Test() 함수 호출
-> 지역 변수 Player player 생성
-> 생성자 Player() 호출, "생성" 출력
-> Test() 함수 실행 종료
-> player가 스코프를 벗어남
-> 소멸자 ~Player() 호출, "소멸" 출력
```

출력 순서는 다음과 같습니다.

```text
생성
소멸
```

## 스택 객체와 힙 객체

스택에 만든 객체는 스코프를 벗어나면 소멸자가 자동으로 호출됩니다.

```cpp
void Test()
{
    Player player;
}
```

힙에 만든 객체는 `delete`를 호출해야 소멸자가 호출됩니다.

```cpp
Player* player = new Player(); // 생성자 호출
delete player;                 // 소멸자 호출
```

`delete`를 하지 않으면 소멸자가 호출되지 않아 자원 정리가 제대로 되지 않을 수 있습니다.

## 면접 답변

생성자는 객체가 생성될 때 자동으로 호출되어 초기화 작업을 수행하는 함수입니다. 소멸자는 객체가 소멸될 때 자동으로 호출되어 동적 할당 메모리 해제나 파일 닫기 같은 자원 정리를 수행합니다. 스택 객체는 스코프를 벗어나면 소멸자가 자동 호출되고, 힙 객체는 `delete`를 호출해야 소멸자가 실행됩니다.
