# 메모리와 주소

## 한 줄 정의

메모리는 데이터를 저장하는 공간이고, 주소는 그 데이터가 저장된 위치를 나타내는 값입니다.

## 왜 필요한가

C++에서 포인터를 이해하려면 먼저 메모리와 주소를 알아야 합니다.

포인터는 결국 "주소를 저장하는 변수"이기 때문에, 주소가 무엇인지 모르면 포인터도 문법처럼만 외우게 됩니다.

## 핵심 설명

예를 들어 아래 코드가 있다고 해봅시다.

```cpp
int hp = 100;
```

사람은 `hp`라는 이름을 보고 이해합니다.

하지만 컴퓨터가 실제로 데이터를 다룰 때는 변수명 자체보다 메모리 위치, 즉 주소를 기준으로 접근합니다.

입문 단계에서는 이렇게 생각하면 좋습니다.

```text
hp
-> 메모리의 어떤 위치
-> 그 위치에 100 저장
```

즉 변수는 값을 담는 이름이기도 하지만, 더 정확히는 어떤 메모리 공간에 붙인 이름이라고 볼 수 있습니다.

## 주의할 점

"CPU가 항상 변수명을 고정 주소 하나로 바꾼다"라고 이해하면 너무 단순합니다.

실제로는 컴파일러가 상황에 따라 레지스터, 스택 위치, 메모리 주소 등 CPU가 실행할 수 있는 형태로 바꿉니다.

다만 처음 배울 때는 "변수명은 사람이 보는 이름이고, 실행 단계에서는 메모리 위치를 기준으로 다룬다" 정도로 이해하면 충분합니다.

## C++ 연결

주소를 직접 확인할 때는 `&` 연산자를 사용합니다.

```cpp
#include <iostream>
using namespace std;

int main()
{
    int hp = 100;

    cout << hp << endl;   // 값
    cout << &hp << endl;  // 주소
    cout << sizeof(hp) << endl; // 크기
}
```

## Unity 연결

Unity의 C#에서는 포인터를 직접 자주 쓰지는 않지만, 참조 타입 개념은 계속 나옵니다.

예를 들어 `Player player;`는 객체 자체를 통째로 들고 있다기보다, 힙에 있는 `Player` 객체를 참조한다고 이해하면 좋습니다.

그래서 메모리와 주소를 이해하면 Reference Type, GC, Object Pooling도 더 쉽게 이어집니다.

## Unreal 연결

Unreal C++에서는 포인터가 자주 등장합니다.

```cpp
AActor* Player;
```

여기서 `*`는 `Player`가 `AActor` 객체의 주소를 저장할 수 있는 포인터 변수라는 뜻입니다.

## 면접 답변

메모리 주소는 데이터가 저장된 메모리상의 위치를 나타내는 값입니다. 변수명은 사람이 코드를 이해하기 쉽게 붙인 이름이고, 프로그램이 실행될 때는 컴파일러가 이를 CPU가 접근할 수 있는 형태로 바꿉니다.
