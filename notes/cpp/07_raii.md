# RAII

## 한 줄 정의

RAII는 자원의 획득과 해제를 객체의 생성자와 소멸자에 묶는 C++의 자원 관리 방식입니다.

## 왜 필요한가

직접 `new/delete`를 사용하면 사람이 실수할 수 있습니다.

```text
delete를 빼먹음 -> 메모리 누수
delete 후 다시 접근 -> 댕글링 포인터
두 번 delete -> 위험
```

RAII는 이런 실수를 줄이기 위해 자원의 수명을 객체의 수명에 묶습니다.

```text
객체 생성 -> 자원 획득
객체 소멸 -> 자원 해제
```

## 생성자/소멸자와 연결

RAII는 생성자와 소멸자를 이용합니다.

```text
생성자
-> 자원을 얻음

소멸자
-> 자원을 정리함
```

스택 객체는 스코프를 벗어나면 소멸자가 자동으로 호출됩니다.

RAII는 이 자동 소멸 타이밍을 이용해서 자원을 자동으로 정리합니다.

## 직접 new/delete의 문제

```cpp
void Test()
{
    int* value = new int(10);

    // delete를 까먹으면 메모리 누수
}
```

이 코드는 `delete`를 직접 작성해야 합니다.

작업량이 많아지거나 예외, 조건문, 중간 return이 많아지면 `delete`를 빼먹기 쉽습니다.

## 스마트 포인터

스마트 포인터는 RAII 원리를 이용해 동적 메모리를 자동으로 관리해주는 도구입니다.

```cpp
#include <memory>

void Test()
{
    std::unique_ptr<int> value = std::make_unique<int>(10);
}
```

`Test()` 함수가 끝나면 `value`가 스코프를 벗어납니다.

그러면 `unique_ptr`의 소멸자가 자동으로 호출되고, 내부에서 자신이 가진 힙 메모리를 정리합니다.

따라서 직접 `delete`를 작성하지 않아도 됩니다.

## RAII와 스마트 포인터의 관계

```text
RAII
-> 자원 관리를 객체 수명에 묶는 방식

스마트 포인터
-> RAII 원리를 메모리 관리에 적용한 도구
```

즉 "스마트 포인터인 RAII"가 아니라, "RAII 원리를 이용한 스마트 포인터"라고 표현하는 것이 더 정확합니다.

## RAII가 관리할 수 있는 자원

RAII는 메모리뿐 아니라 여러 자원에 사용할 수 있습니다.

```text
동적 메모리
파일 핸들
네트워크 연결
뮤텍스 락
그래픽 리소스
```

핵심은 자원을 직접 열고 닫는 일을 사람이 기억하게 하지 않고, 객체 수명에 맡기는 것입니다.

## 게임 개발 연결

게임 개발에서는 리소스 관리가 중요합니다.

```text
메모리
텍스처
사운드
파일
네트워크 연결
엔진 객체
```

자원을 직접 관리하다 보면 누수나 해제 순서 문제가 생기기 쉽습니다.

C++에서는 RAII와 스마트 포인터를 이해하면 자원 수명 관리의 기본 감각을 잡을 수 있습니다.

## 면접 답변

RAII는 자원의 획득과 해제를 객체의 생성자와 소멸자에 묶는 C++의 자원 관리 방식입니다. 객체가 생성될 때 자원을 획득하고, 객체가 스코프를 벗어나 소멸될 때 소멸자에서 자동으로 자원을 해제하게 만들어 메모리 누수 같은 실수를 줄일 수 있습니다. 스마트 포인터는 이 RAII 원리를 이용해 동적 메모리를 자동으로 관리해주는 도구입니다.
