# 클래스와 객체

## 한 줄 정의

클래스는 객체를 만들기 위한 설계도이고, 객체는 그 설계도로 실제 만들어진 실체입니다.

## 왜 필요한가

게임 개발에서는 플레이어, 몬스터, 총알, 아이템, 스킬 같은 요소를 코드로 표현해야 합니다.

클래스와 객체를 사용하면 데이터와 행동을 하나의 단위로 묶어서 관리할 수 있습니다.

## 클래스

클래스는 객체를 만들기 위한 설계도입니다.

```cpp
class Player
{
public:
    int hp;
    int mp;

    void Attack()
    {
        cout << "Attack" << endl;
    }
};
```

이 코드는 아직 실제 플레이어를 만든 것이 아닙니다.

`Player`는 이런 구조를 가진 객체를 만들 수 있다는 설계도입니다.

```text
Player는 hp와 mp를 가지고,
Attack 기능을 가진다.
```

## 객체

객체는 클래스를 바탕으로 실제 만들어진 것입니다.

```cpp
Player player;
```

이제 `player`라는 실제 객체가 만들어졌습니다.

```cpp
player.hp = 100;
player.mp = 50;
player.Attack();
```

```text
Player 클래스 = 설계도
player 객체 = 실제 만들어진 플레이어
```

## 같은 클래스로 여러 객체 만들기

같은 클래스로 여러 객체를 만들 수 있습니다.

```cpp
Player player1;
Player player2;

player1.hp = 100;
player2.hp = 50;
```

`player1`과 `player2`는 같은 `Player` 클래스로 생성된 객체지만, 각각 독립된 메모리 공간에 자신의 멤버 변수를 가지고 있습니다.

따라서 서로 다른 값을 가질 수 있습니다.

```text
player1.hp -> 100
player2.hp -> 50
```

## 멤버 변수와 멤버 함수

클래스 안에 있는 변수를 멤버 변수라고 합니다.

```cpp
int hp;
int mp;
```

클래스 안에 있는 함수를 멤버 함수라고 합니다.

```cpp
void Attack();
```

정리하면 다음과 같습니다.

```text
멤버 변수 = 객체가 가지는 데이터
멤버 함수 = 객체가 할 수 있는 행동
```

게임으로 보면 다음처럼 생각할 수 있습니다.

```text
Player
데이터: hp, mp, position
행동: Attack(), Move(), TakeDamage()
```

## 객체마다 데이터는 따로, 함수 코드는 공유

```cpp
Player player1;
Player player2;
```

`player1.hp`와 `player2.hp`는 서로 다른 값을 가질 수 있습니다.

하지만 `Attack()` 함수 코드 자체가 객체마다 복사되어 따로 생기는 것은 아닙니다.

개념적으로는 다음처럼 이해하면 좋습니다.

```text
객체마다 멤버 변수는 따로 있음
멤버 함수 코드는 클래스 단위로 공유됨
```

같은 `Attack()` 함수 코드라도 `player1.Attack()`을 호출하면 `player1`의 데이터를 기준으로 동작하고, `player2.Attack()`을 호출하면 `player2`의 데이터를 기준으로 동작합니다.

이것을 가능하게 해주는 개념이 `this` 포인터입니다.

## 인스턴스

객체를 인스턴스라고 부르기도 합니다.

```text
객체 = 인스턴스
```

조금 더 정확히 말하면 다음과 같습니다.

```text
player는 Player 클래스의 인스턴스다.
```

즉 `Player` 설계도로 실제 만들어진 하나의 실체라는 뜻입니다.

## 게임 개발 연결

게임에서는 클래스와 객체 개념이 계속 등장합니다.

```text
Player 클래스
-> 실제 플레이어 객체

Monster 클래스
-> 실제 몬스터 객체들

Bullet 클래스
-> 발사된 총알 객체들

Item 클래스
-> 필드에 떨어진 아이템 객체들
```

Unreal에서도 마찬가지입니다.

```text
ACharacter 클래스
-> 실제 월드에 배치된 캐릭터 액터

AActor 클래스
-> 월드에 존재하는 액터 객체
```

Unity에서도 비슷하게 볼 수 있습니다.

```text
Enemy 스크립트 클래스
-> 씬에 붙어 있는 실제 Enemy 컴포넌트 객체
```

## 면접 답변

클래스는 객체를 만들기 위한 설계도이고, 객체는 그 클래스를 바탕으로 메모리에 실제 생성된 실체입니다. 클래스에는 멤버 변수와 멤버 함수가 정의되고, 객체는 각자 자신의 멤버 변수 값을 가질 수 있습니다. 같은 클래스로 여러 객체를 만들 수 있으며, 각 객체는 서로 다른 상태를 가질 수 있습니다.

## 확인 질문

- 멤버 변수와 멤버 함수는 각각 무엇일까?
- 객체를 인스턴스라고 부르는 이유는 무엇일까?
